# R1.5-10 QA Check (DoD + AC Traceability)

Story: **R1.5-10 – Adaptive vs. fixe Schwellen im Profil konfigurieren**

## AC-Umsetzung
1. **AC1 – Modus Fix/Adaptiv pro Schwelle im Profil**
   - Frontend ergänzt je Schwellenfeld eine Modusauswahl (`Fixed`/`Adaptive`) in den Profileinstellungen.
   - Backend erweitert das Profilmodell um persistierte Modusfelder je Schwelle.
   - Abgedeckt durch Tests:
     - `R1_5_10_Ac01_Ac03_shows_and_saves_threshold_modes_with_adaptive_source_visibility`
     - `R1_5_10_Ac01_Ac04_UpdateProfile_ShouldPersistThresholdModesAndRejectInvalidMode`

2. **AC2 – Adaptiver Modus als Maximum über alle Sessions**
   - Backend berechnet bei Upload/Rekalkulation für adaptive Schwellen den wirksamen Wert als Maximalwert über alle Sessions des Nutzers.
   - Datenbasis basiert auf den bereits gespeicherten Session-Rohdaten.

3. **AC3 – Wirksamer Wert inkl. Quelle transparent sichtbar**
   - API liefert je Schwelle neben dem Wert auch Mode/Source (`Fixed`/`Adaptive`) in `summary.coreMetrics.thresholds`.
   - UI zeigt die wirksamen Schwellenwerte inkl. Source in der Session-Ansicht.

4. **AC4 – Validierung/Hinweise gegen widersprüchliche Konfiguration**
   - Backend verhindert widersprüchliche Kombi (Sprint + High-Intensity beide adaptiv).
   - UI zeigt Datenbasis-Hinweis für adaptive Modi.
